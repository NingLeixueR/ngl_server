# makeproto 协议生成工具

`server/makeproto` 是面向协议和数据库生成链路的本地工具模块。

## 构建方式

和 `idl` 一样，`makeproto` 当前主要在本地 Windows 构建中启用，不是主 CI 路径上的默认构建目标。

## 主要文件

- `core/main.cpp`
  工具入口和主流程组织
- `core/xml_protocol.h`
  协议生成逻辑主体
- `core/xml_protocol.cpp`
  协议生成实现入口

## 模块职责

- 根据 `proto` 输入生成协议 XML 或映射信息
- 参与数据库建表 SQL 等辅助产物生成
- 为服务端和工具链提供一致的协议描述来源

## 与其他模块的关系

- 上游依赖
  `public/cpp` 与 `nglcore` 暴露的基础能力
- 下游影响
  协议描述文件、映射文件、生成 SQL 等产物

## 常见改动入口

- 修改协议 XML 结构
  看 `xml_protocol.h`
- 修改工具入口参数或执行流程
  看 `main.cpp`
- 修改生成模板和输出格式
  看生成逻辑主类

## 维护建议

- 生成规则发生变更时，建议同步更新文档和样例输出。
- 工具输出应尽量可重复，避免同一输入在不同机器上出现无意义差异。
- 如果未来要让前后端共用协议描述，优先稳定 `makeproto` 的产物格式。
